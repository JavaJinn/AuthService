Protected Area

Welcome, User

This page proves that registration, login, session handling and route protection are connected.

What already works

  • Flyway creates the auth tables automatically.
  • New users are stored in PostgreSQL.
  • Passwords are hashed before saving.
  • Spring Security protects the dashboard.

Good next project steps

  • Add email verification.
  • Introduce DTO-to-entity mapping tests.
  • Create profile and settings pages.
  • Add role-based admin pages later.

Portfolio angle

This setup is intentionally clean and easy to explain in an interview: configuration is externalized, schema changes are versioned, authentication is form-based, and the UI shows a complete user flow from signup to login to protected content.
